API-INCOME

В даному АРІ тесті перевірялася можливість додавання нового доходу за допомогою АРІ для залогіненого користувача
Був викоританий JEST фреймфорк для тесту

При запуску тесту в терміналі були передані 3 параметри
X-Access-Token
X-Username
X-Refresh-Token

Після чого відбувався запуск командою
npx jest tests/api-income.spec.ts -t "should create an income and return success message with ID"
(можна було без -t "should create an income and return success message with ID" так як тест лише один, лишив як демонстрацію)

токени динамічні, їх кожен раз треба передавати нові

FOP-INCOME

В даному UI тесті було перевірена можливість додавання нового income із перевіркою на успішність

Спочатку був прописаний beforeEach для логіну перед кожним тестом,
оскільки без нього додати новий дохід який буде відображено було б неможливо

Потім було створено декілька малих тестів, такі як додати окремо коментар, обрати валюту чи вказати дату
а в кінці основний тест із використанням вище описаних перевірок

P.S.: На відео останній тест фейлиться, проте це помилка програми, адже замість тосту про успішне додавання
відбувається помилка на стороні сервісу
